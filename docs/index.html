The Skill-Building Workflow

The repository is built around transparent execution: define the task, configure the runtime, generate assets or structured output, then package the result for real-world use.

Product UI
Codex CLI
📄
1. Define

Specify a reusable workflow in a clear SKILL.md file with constraints, prompts, and output expectations.

→
⚙️
2. Configure

Pick the runtime path: native image tool, OpenAI API, Gemini Web, or a utility workflow like publishing or conversion.

→
▶️
3. Execute

Run the skill from Codex CLI or another compatible agent runtime and generate the working files or media outputs.

→
🔗
4. Package

Merge outputs into final artifacts such as markdown, publishable visuals, or deck files like PPTX and PDF.
